## COUNT

> Liczy ilość wybranych rekordów

``` sql
 SELECT COUNT( column_name/ * ) FROM table_name WHERE condition; 
 ```

 - Alias
 > Stwarza alias dla wybranych rekordów

``` sql
 SELECT COUNT( column_name/ * ) AS ["alias"] FROM table_name WHERE condition; `*
```

 - GROUP BY
> Grupuje rekordy wybraną kolumną

``` sql
 SELECT COUNT( * ) AS ["alias"] FROM table_name WHERE condition GROUP BY column_name; 
```

## LIKE

> Wybiera rekord który mieści wybrany znak lub słowo

``` sql
 SELECT * FROM table_name WHERE condition LIKE condition; 
```

- % reprezentuje 0, 1 albo kilku znaków, np:
**"... LIKE 'ab%';"** wyszuka w wybranej kolumnie rekord, który zaczyna się na "ab";
- _ reprezentuje jeden i tylko jeden znak, np:
**"... LIKE 'Ab_d__g';"** wyszuka w wybranej kolumnie rekord który ma te znaki  w tych miejscach;

(P.S., '%a%' wyszuka rekord, który zawiera znak "a");

## BETWEEN

> Wybiera rekordy z wartością w wybranych ramkach

``` sql
 SELECT * FROM table_name WHERE 'condition' BETWEEN value1 AND value2; 
```

- NOT BETWEEN

> Wybiera rekordy z wartością poza wybranymi ramkami

``` sql
 SELECT * FROM table_name WHERE 'condition' NOT BETWEEN value1 AND value2; 
```

- IN

>Pozwala wybrać między wieloma warościami

``` sql
 SELECT * FROM table_name WHERE 'condition' IN (1,2,3); 
```

(P.S., jeżeli wartości BETWEEN są tekstowymi, wybiera rekordy między tymi wartościami w alphabetycznym porządku)
(P.S 2., żeby wybrać rekordy w ramkach którejść daty, trzeba ją zapisać w hasztagach **#11/02/2026#** lub w pojedynczych cudzysłowiach **'2026-02-11'**)

## JOIN

> Kombinuje rekordy z dwoch lub więcej tabel za pomocą kolumny która ma relacje we wszystkich tabelach

``` sql
SELECT Table1.Column, Table2.Column FROM Table1 INNER JOIN Table2 ON Table1.Relation=Table2.Relation;
```

- INNER JOIN: zwarca rekordy które mają pasujące wartości w oboch tabelach;
- LEFT JOIN: zwraca wszystkie rekordy z lewej tabeli (table1), i pasujące rekordy z prawej;
- RIGHT JOIN: zwraca wszystkie rekordy z prawej tabeli (table2), i pasujące rekordy z lewej;
- FULL JOIN: Zwraca wszystkie rekordy, jeśli pasują do siebie w tabeli lewej lub prawej

## HAVING

> Filtruje rezulty działania GROUP BY

```sql 
SELECT column1, aggregate_function(column2), column3, ...
FROM table_name
WHERE condition
GROUP BY column1, column3
HAVING condition -- The condition on grouped data
ORDER BY column_name;
```
